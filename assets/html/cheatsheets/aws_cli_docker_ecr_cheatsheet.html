Quick Reference · AWS CLI v2 · Docker CLI · Amazon ECR

aws cli cheat sheet

Every AWS CLI call follows the same pattern: aws <service> <action> [options]. Also covers Docker CLI (build → run → debug → clean) and Amazon ECR (authenticate → push → scan → lifecycle). Master the global flags (--output, --query, --profile, --region) and you can drive any of 200+ services from the terminal.

setup & config S3 storage compute (EC2 · Lambda · ECS) IAM & security databases (RDS · DynamoDB) networking (Route53 · CF · ELB) ops (CloudWatch · SNS · SQS) Docker CLI ECR (container registry) ★ most common

Cross-referenced from: docs.aws.amazon.com/cli/latest · devhints.io/awscli · bluematador.com/learn/aws-cli-cheatsheet · gist.github.com/apolloclark · pluralsight.com S3 cheatsheet

Command anatomy — every service, every action, same shape
01Install & Setupfirst-time
  • pip install awscli★
    Install AWS CLI v1 via pip (Python package manager).
  • brew install awscli★
    Install on macOS via Homebrew — keeps updates easy.
  • aws --version
    Confirm the installed version.
  • aws configure★
    Interactive wizard: sets Access Key, Secret, Region, and output format in ~/.aws/credentials & ~/.aws/config.
  • aws configure --profile prod★
    Create a named profile — useful for multiple accounts or roles.
  • aws configure list
    Show current active credentials & config source.
  • aws configure list-profiles
    List all locally configured profiles.

Credentials precedence: env vars → CLI flags → ~/.aws/credentials → IAM instance role. An instance role is the recommended production approach — no local keys needed.

02Global Flagswork with every service
  • aws s3 ls --profile prod★
    Use a non-default named profile for one command.
  • aws ec2 describe-instances --region eu-west-1★
    Override the region for one command.
  • aws ec2 describe-instances --output table★
    Output as human-readable table (also: json, yaml, text).
  • aws ec2 describe-instances --query 'Reservations[].Instances[].[InstanceId,State.Name]'★
    JMESPath query to pluck specific fields from JSON output.
  • aws ec2 run-instances ... --dry-run
    Check permissions & validate params without making real changes.
  • aws ec2 describe-instances help★
    Append help to any (sub-)command for inline docs.
  • aws sts get-caller-identity★
    Show which IAM user / role is currently authenticated.
03Filtering Output with jqparse JSON like a pro
  • aws ec2 describe-instances | jq -r '.Reservations[].Instances[] | .InstanceId'★
    jq -r pipes JSON → extracts just the instance IDs as plain text.
  • aws iam list-users | jq -r '.Users[] | .UserId+" "+.UserName'★
    Concatenate fields with + for space-separated output.
  • aws dynamodb scan --table-name events --select count | jq .ScannedCount
    Extract a single top-level value.
  • ... | jq -r '.Items[] | select(.Status == "ACTIVE") | .Name'
    select() filters array elements by a condition.

Tip: install jq with brew install jq or apt install jq. It turns verbose JSON into scriptable one-liners.

04S3 — Bucketss3 · s3api
  • aws s3 ls★
    List all buckets in the account.
  • aws s3 ls s3://mybucket/★
    List objects (files) inside a bucket or prefix.
  • aws s3 mb s3://my-new-bucket★
    Make (create) a new bucket — name must be globally unique.
  • aws s3 rb s3://my-bucket --force
    Remove bucket and all contents. --force deletes non-empty buckets.
  • aws s3api get-bucket-acl --bucket my-bucket
    Check Access Control List — is it public? (low-level s3api command).
  • aws s3api put-bucket-acl --acl private --bucket my-bucket
    Make a bucket private (revoke public access).
  • aws s3api put-bucket-versioning --bucket my-bucket --versioning-configuration Status=Enabled
    Enable object versioning — keeps old versions on overwrite/delete.
05S3 — Objectscp · mv · rm · sync · presign
  • aws s3 cp backup.tar s3://my-bucket/★
    Upload a local file to a bucket.
  • aws s3 cp s3://my-bucket/backup.tar ./★
    Download an object to the current directory.
  • aws s3 cp myfolder/ s3://my-bucket/folder/ --recursive★
    Upload an entire local folder. --recursive walks subdirectories.
  • aws s3 mv s3://bucket/old.txt s3://bucket/new.txt
    Move (rename) an object within or between buckets.
  • aws s3 rm s3://my-bucket/secret.gz★
    Delete a single object.
  • aws s3 rm s3://my-bucket/folder/ --recursive
    Delete all objects under a prefix.
  • aws s3 sync ./local-dir/ s3://my-bucket/★
    Sync local → bucket; only uploads new or changed files.
  • aws s3 sync s3://src-bucket/ s3://dst-bucket/
    Sync between two buckets (cross-region backup).
  • aws s3 sync ./ s3://my-bucket/ --exclude '*.tmp'
    Exclude files matching a glob from the sync operation.
  • aws s3 presign s3://bucket/file.pdf --expires-in 3600★
    Generate a temporary signed URL for private objects. Expires in 3600 seconds (1 hour).

s3 vs s3api: s3 = high-level shortcuts (cp, sync, mb). s3api = raw API access for policies, ACLs, versioning.

06EC2 — Instancesvirtual machines
  • aws ec2 describe-instances★
    List all instances (running and stopped) in the region.
  • aws ec2 describe-instances --filters Name=instance-state-name,Values=running★
    Show only running instances.
  • aws ec2 describe-instances | jq -r '.Reservations[].Instances[] | .InstanceId+" "+.InstanceType+" "+(.Tags[]|select(.Key=="Name").Value)'
    Human-readable table of ID, type, and Name tag via jq.
  • aws ec2 run-instances --image-id ami-0abcdef1234 --instance-type t3.micro --security-group-ids sg-xxxx --key-name MyKey★
    Launch (start) a new EC2 instance.
  • aws ec2 start-instances --instance-ids i-1234567890abcdef0★
    Start a stopped instance (billing resumes).
  • aws ec2 stop-instances --instance-ids i-1234567890abcdef0★
    Stop a running instance (billing pauses for compute).
  • aws ec2 terminate-instances --instance-ids i-1234567890abcdef0
    ⚠ Permanently delete an instance and its ephemeral storage.
  • aws ec2 describe-instance-status --instance-ids i-xxx
    Check health checks and system status for an instance.
  • aws ec2 describe-instances --query 'Reservations[].Instances[].[PublicIpAddress,Tags[?Key==`Name`].Value|[0]]' --output text
    List running instance IPs + Name tags, clean text output.
07EC2 — Security Groups & VPCfirewall rules
  • aws ec2 describe-security-groups | jq -r '.SecurityGroups[] | .GroupId+" "+.GroupName'★
    List all security groups with ID and name.
  • aws ec2 create-security-group --group-name web-sg --description "Web access" --vpc-id vpc-xxx
    Create a new security group inside a VPC.
  • aws ec2 authorize-security-group-ingress --group-id sg-xxx --protocol tcp --port 443 --cidr 0.0.0.0/0★
    Open inbound port 443 (HTTPS) from anywhere. Use /0 for all IPs.
  • aws ec2 authorize-security-group-ingress --group-id sg-xxx --protocol tcp --port 22 --cidr $MY_IP/32
    Restrict SSH (port 22) to your own IP only — use /32 for one IP.
  • aws ec2 revoke-security-group-ingress --group-id sg-xxx --protocol tcp --port 22 --cidr 0.0.0.0/0
    Remove an inbound rule from a security group.
  • aws ec2 describe-vpcs | jq -r '.Vpcs[] | .VpcId+" "+.CidrBlock'
    List VPCs and their CIDR blocks.
  • aws ec2 describe-subnets --filter Name=vpc-id,Values=vpc-xxx
    List subnets within a specific VPC.
08EC2 — Key Pairs & AMIsSSH keys · machine images
  • aws ec2 describe-key-pairs
    List all registered SSH key pairs.
  • aws ec2 create-key-pair --key-name MyKey --output text★
    Create a key pair — save the output as a .pem file immediately.
  • aws ec2 import-key-pair --key-name MyKey --public-key-material fileb://~/.ssh/id_rsa.pub
    Import your existing local public key so AWS knows it.
  • aws ec2 delete-key-pair --key-name MyKey
    Remove a key pair from AWS (doesn't affect local files).
  • aws ec2 describe-images --filters Name=is-public,Values=false --query 'Images[].[ImageId,Name]' --output text★
    List your private AMIs (machine images).
  • aws ec2 create-image --instance-id i-xxx --name "MyAMI-v1" --no-reboot
    Snapshot a running instance into an AMI without rebooting.
  • aws ec2 deregister-image --image-id ami-xxx
    Delete an AMI (doesn't delete the backing EBS snapshots).
  • aws ec2 create-tags --resources i-xxx --tags Key=Env,Value=prod
    Add or update tags on any resource — tags are key–value labels.
09IAM — Users & Access Keysidentity
  • aws iam list-users | jq -r '.Users[] | .UserId+" "+.UserName'★
    List all IAM users in the account.
  • aws iam get-user
    Show the current caller's IAM user info.
  • aws iam create-user --user-name alice★
    Create a new IAM user (no console access or keys yet).
  • aws iam delete-user --user-name alice
    Delete a user (must remove keys, policies, group memberships first).
  • aws iam list-access-keys --user-name alice★
    List access key IDs for a user (secrets are never shown again after creation).
  • aws iam create-access-key --user-name alice | jq -r '.AccessKey | .AccessKeyId+" "+.SecretAccessKey'★
    Generate a new key pair — save the secret immediately, it won't appear again.
  • aws iam update-access-key --user-name alice --access-key-id AKIA… --status Inactive
    Deactivate a key without deleting it (useful during key rotation).
  • aws iam delete-access-key --user-name alice --access-key-id AKIA…
    Permanently delete an access key.
  • aws iam get-account-password-policy
    View the account-level password complexity rules.
  • aws iam update-account-password-policy --minimum-password-length 12 --require-symbols --require-numbers
    Enforce a stricter password policy for the whole account.
10IAM — Groups, Policies & Rolespermissions
  • aws iam list-groups | jq -r .Groups[].GroupName★
    List all IAM groups — groups collect users and assign shared policies.
  • aws iam create-group --group-name Developers
    Create a new group.
  • aws iam add-user-to-group --group-name Developers --user-name alice★
    Add a user to a group (they inherit the group's policies).
  • aws iam remove-user-from-group --group-name Developers --user-name alice
    Remove a user from a group.
  • aws iam list-policies --scope Local | jq -r '.Policies[] | .PolicyName+" "+.Arn'★
    List custom (non-AWS-managed) policies in the account.
  • aws iam attach-group-policy --group-name Developers --policy-arn arn:aws:iam::aws:policy/AmazonS3FullAccess★
    Attach an AWS managed policy to a group.
  • aws iam detach-group-policy --group-name Developers --policy-arn arn:aws:iam::aws:policy/AmazonS3FullAccess
    Remove a policy from a group.
  • aws iam list-entities-for-policy --policy-arn arn:aws:iam::…:policy/MyPolicy
    See which users/groups/roles use a given policy.
  • aws iam list-roles | jq -r '.Roles[] | .RoleName'
    List IAM roles — roles are assumed by services, instances, or users.

13DynamoDBNoSQL key–value & document store
  • aws dynamodb list-tables | jq -r .TableNames[]★
    List all DynamoDB tables.
  • aws dynamodb scan --table-name events★
    Return all items from a table (expensive on large tables — use for dev/debug).
  • aws dynamodb scan --table-name events --select count | jq .ScannedCount
    Count items without fetching their data.
  • aws dynamodb get-item --table-name events --key '{"email":{"S":"bob@example.com"}}'★
    Fetch one item by its primary key. Note the DynamoDB type notation ("S" = String).
  • aws dynamodb put-item --table-name events --item '{"email":{"S":"bob@example.com"},"age":{"N":"30"}}'
    Insert or overwrite one item.
  • aws dynamodb delete-item --table-name events --key '{"email":{"S":"bob@example.com"}}'
    Delete one item by primary key.
  • aws dynamodb query --table-name events --key-condition-expression "email = :e" --expression-attribute-values '{":e":{"S":"bob@example.com"}}'
    Query items by partition key (efficient — uses the index).

16Route 53DNS management
  • aws route53 list-hosted-zones | jq -r '.HostedZones[] | .Id+" "+.Name'★
    List all hosted zones (DNS domains) in the account.
  • aws route53 create-hosted-zone --name example.com --caller-reference $(date +%s)
    Create a new hosted zone. caller-reference is a unique ID to prevent duplicate creation.
  • aws route53 list-resource-record-sets --hosted-zone-id Z1234★
    List all DNS records (A, CNAME, MX, etc.) in a zone.
  • aws route53 change-resource-record-sets --hosted-zone-id Z1234 --change-batch file://records.json★
    Create / update / delete records via a JSON change-batch file.
  • aws route53 delete-hosted-zone --id Z1234
    Delete a hosted zone (must be empty first).

Change-batch: use Action CREATE, UPSERT (create or update), or DELETE in the JSON file.

20Docker — Imagesbuild · tag · inspect
  • docker build -t myapp:1.0 .★
    Build an image from the Dockerfile in the current dir; -t gives it a name:tag.
  • docker build -t myapp:1.0 . --no-cache
    Force a full rebuild — skips all layer cache. Use after changing base images.
  • docker build -f Dockerfile.prod -t myapp:prod .
    Build from a non-default Dockerfile name.
  • docker images★
    List all local images with repository, tag, ID, and size.
  • docker tag myapp:1.0 myrepo/myapp:latest★
    Add a second tag to an existing image — required before pushing to a registry.
  • docker pull nginx:latest★
    Download an image from Docker Hub (or any registry) to your machine.
  • docker push myrepo/myapp:1.0★
    Upload an image to a registry. Must docker login first.
  • docker inspect myapp:1.0
    Full JSON metadata: layers, env vars, entrypoint, exposed ports, size.
  • docker history myapp:1.0
    Show each layer of an image and the command that created it.
  • docker save -o myapp.tar myapp:1.0
    Export an image to a tar file for offline transfer or backup.
  • docker load -i myapp.tar
    Import an image from a tar file produced by docker save.
  • docker rmi myapp:1.0
    Delete a local image. Fails if a container is using it — stop it first.
  • docker image prune
    Remove all dangling images (untagged layers with no container using them).
  • docker image prune -a
    Remove all unused images — including tagged ones with no running container.
21Docker — Containersrun · manage · lifecycle
  • docker run -d -p 8080:80 --name web nginx★
    -d detached (background), -p host:container port mapping, --name gives a friendly name.
  • docker run -it --rm ubuntu /bin/bash★
    -it interactive+TTY terminal, --rm auto-deletes container on exit.
  • docker run -d -e DB_HOST=localhost -v ./data:/var/data myapp★
    -e sets env var, -v host-path:container-path mounts a volume.
  • docker ps★
    List running containers with ID, name, status, and ports.
  • docker ps -a★
    List all containers — including stopped ones.
  • docker start web / docker stop web★
    Start or stop an existing container by name or ID.
  • docker restart web
    Stop then start a container — same as stop + start in one step.
  • docker rm web
    Remove a stopped container. Add -f to force-remove a running one.
  • docker container prune
    Remove all stopped containers in one shot.
  • docker container stats
    Live CPU, memory, network, and I/O usage for all running containers.
22Docker — Exec & Debugshell in · logs · copy files
  • docker exec -it web /bin/bash★
    Open an interactive shell inside a running container — the most common debug tool.
  • docker exec -it web sh
    Use sh if bash is not installed (common in alpine-based images).
  • docker exec web cat /etc/hosts
    Run a one-off command inside a running container without opening a shell.
  • docker logs -f web★
    Tail & follow a container's stdout/stderr. -f = follow (like tail -f).
  • docker logs web --tail 100
    Show only the last 100 log lines.
  • docker logs web --since 30m
    Show logs from the last 30 minutes.
  • docker inspect web★
    Full JSON state of a container — IP, mounts, env vars, restart policy.
  • docker inspect web | jq -r '.[0].NetworkSettings.IPAddress'
    Extract a specific field — container's internal IP address.
  • docker cp web:/app/config.json ./config.json
    Copy a file from a container to your host. Reverse args to copy in.
  • docker port web
    Show which host ports are mapped to which container ports.
  • docker top web
    List processes running inside a container (like ps aux inside).
  • docker diff web
    Show filesystem changes made inside a container since it started.
23Docker — Volumes & Networkspersistent data · container DNS
  • docker volume create mydata★
    Create a named volume — survives container removal; managed by Docker.
  • docker volume ls
    List all volumes on the host.
  • docker volume inspect mydata
    Show where the volume is stored on disk and which containers use it.
  • docker run -d -v mydata:/var/lib/postgresql/data postgres★
    Mount a named volume into a container — data persists after the container stops.
  • docker volume rm mydata
    Delete a volume and all its data permanently.
  • docker network create mynet★
    Create a user-defined bridge network — containers on the same network find each other by name.
  • docker network ls
    List all Docker networks (bridge, host, none, custom).
  • docker run -d --network mynet --name api myapp
    Attach a container to a network at start. It can reach other containers by their names.
  • docker network connect mynet web
    Attach a running container to an additional network.
  • docker network inspect mynet
    Show subnet, gateway, and which containers are connected.

DNS tip: containers on a user-defined network resolve each other by container name. E.g. api can reach db at http://db:5432.

24Compose & Cleanupmulti-container · prune
  • docker compose up -d★
    Start all services defined in docker-compose.yml in detached mode.
  • docker compose up --build
    Rebuild images before starting — use after changing a Dockerfile.
  • docker compose down★
    Stop and remove containers, networks defined in the compose file.
  • docker compose down -v
    Also removes named volumes declared in the compose file.
  • docker compose ps
    List service containers and their status.
  • docker compose logs -f api
    Follow logs from a specific service.
  • docker compose exec api /bin/sh
    Shell into a running service container from compose.
  • docker system prune★
    Remove stopped containers, unused networks, dangling images, and build cache.
  • docker system prune -a --volumes
    Nuclear cleanup — also removes all unused images and volumes. Frees most disk space.
  • docker system df
    Show how much disk space Docker is using (images, containers, volumes, cache).
25ECR — Authentication⚠ required before every push/pull
  • aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin 123456789.dkr.ecr.us-east-1.amazonaws.com★
    The canonical ECR login. Pipes a 12-hour token directly into Docker. Replace account ID and region.
  • aws ecr get-login-password --region us-east-1 --profile prod | docker login --username AWS --password-stdin 123456789.dkr.ecr.us-east-1.amazonaws.com
    Same but uses a named AWS profile (multi-account setups).
  • aws ecr-public get-login-password --region us-east-1 | docker login --username AWS --password-stdin public.ecr.aws
    Authenticate to the ECR Public Gallery — must always use us-east-1.

Token lifetime: authorization tokens are valid for 12 hours. In CI/CD pipelines, run the login step fresh at the start of every job.
IAM needed: ecr:GetAuthorizationToken + repo permissions (ecr:BatchGetImage, ecr:PutImage…).

27ECR — Push / Pull Workflowthe 4-step pattern
  • # 1️⃣ Authenticate (12-hour token)
    aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin $ACCOUNT.dkr.ecr.us-east-1.amazonaws.com    ★
    Step 1: get a token and hand it straight to Docker.
  • # 2️⃣ Build locally
    docker build -t my-app:v2 .    ★
    Step 2: build the image with a meaningful local tag.
  • # 3️⃣ Tag for ECR
    docker tag my-app:v2 $ACCOUNT.dkr.ecr.us-east-1.amazonaws.com/my-app:v2    ★
    Step 3: retag with the full ECR URI. ECR URI = <account>.dkr.ecr.<region>.amazonaws.com/<repo>:<tag>.
  • # 4️⃣ Push
    docker push $ACCOUNT.dkr.ecr.us-east-1.amazonaws.com/my-app:v2    ★
    Step 4: upload to ECR. Only changed layers are transferred.
  • # Pull back (from any machine)
    docker pull $ACCOUNT.dkr.ecr.us-east-1.amazonaws.com/my-app:v2    ★
    Pull to any authenticated Docker host (re-auth first).

Shortcut: in the ECR console, open your repo and click View push commands — it generates all 4 steps with your exact account/region filled in.
